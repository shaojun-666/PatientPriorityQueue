# 患者优先排队系统

模拟医院患者排队的数据结构课程项目，使用 **优先队列** 实现患者按优先级就诊的管理系统。

## 项目结构

```
PatientPriorityQueue/
├── MajorHM_2_1_List/      # 多链表实现
│   └── 源.cpp
└── MajorHM_2_1_Heap/      # 二叉堆实现
    └── 源.cpp
```

## 两种实现方式

### 1. 多链表优先队列（MajorHM_2_1_List）

基于 **哈希表 + 双向链表** 实现：

- 每个优先级对应一条双向链表，存储在 `unordered_map<int, LinkedListQueue*>` 中
- 通过 `idToNode` 哈希表实现 O(1) 按 ID 查找
- 支持的操作与堆版本相同，但内部结构不同

### 2. 二叉堆优先队列（MajorHM_2_1_Heap）

基于 **大顶堆（vector 实现）** + 惰性删除：

- 使用 `vector<HeapNode>` 维护堆结构，`idToIndex` 哈希表记录节点在堆中的位置
- 删除节点时采用 **惰性删除**（标记 `deleted = true`），避免频繁堆化带来的性能开销
- 相同优先级时按 **入队时间戳**（FIFO）决定顺序，保证公平性

## 支持的操作

| 操作 | 说明 |
|------|------|
| `enqueue(name, id, priority)` | 添加患者（姓名、ID、优先级） |
| `dequeue()` | 取出优先级最高的患者 |
| `remove(id)` | 按 ID 移除指定患者 |
| `switchQueue` / `updatePriority` | 修改患者的优先级 |
| `print()` | 打印当前队列状态 |

## 运行示例

```
===== 多链表实现优先队列演示 =====

成功添加：Tom (ID: 123, 优先级: 4)
成功添加：Jerry (ID: 456, 优先级: 2)
成功添加：Alice (ID: 789, 优先级: 5)
成功添加：Bob (ID: 111, 优先级: 3)

=== 多链表优先队列当前状态 ===
1. Alice (ID: 789, 优先级: 5)
2. Tom (ID: 123, 优先级: 4)
3. Bob (ID: 111, 优先级: 3)
4. Jerry (ID: 456, 优先级: 2)
===========================

dequeue → Alice（最高优先级 5）
dequeue → Bob（同优先级 5，按时间戳先进先出）
```

## 时间复杂度对比

| 操作 | 多链表 | 二叉堆 |
|------|--------|--------|
| enqueue | O(1) | O(log n) |
| dequeue | O(P)* | O(log n) |
| remove | O(1) | O(1) ** |
| updatePriority | O(1) | O(log n) |

> \* P = 优先级种类数。需要从高到低遍历查找非空队列。  
> \** 惰性删除仅标记，实际节点在 dequeue 时清理。

## 设计要点

- **ID 唯一性**：通过哈希表确保每位患者 ID 唯一
- **惰性删除**（堆版本）：删除时仅标记而不立即移除，出队时跳过已删节点，摊还复杂度仍为 O(log n)
- **优先级更新**（堆版本）：删除旧记录后重新入队新记录，利用惰性删除保持堆结构正确
